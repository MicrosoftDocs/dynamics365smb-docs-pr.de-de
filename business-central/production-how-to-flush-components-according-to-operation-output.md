<span data-ttu-id="e7993-106">Diese Funktionalität ist aus folgenden Ursachen nützlich:</span><span class="sxs-lookup"><span data-stu-id="e7993-106">This functionality is useful for the following reasons:</span></span>  

- <span data-ttu-id="e7993-107">**Lagerwert ermitteln**</span><span class="sxs-lookup"><span data-stu-id="e7993-107">**Inventory Valuation**</span></span>

    <span data-ttu-id="e7993-108">Wertposten für Istmeldungen und Verbräuche werden beim Fortschritt des Fertigungsauftrags erstellt.</span><span class="sxs-lookup"><span data-stu-id="e7993-108">Value entries for output and consumption are created in parallel as the production order progresses.</span></span> <span data-ttu-id="e7993-109">Ohne Verbindungscodes erhöht sich der Lagerwert, wenn Ausstoß registriert wird, und vermindert sich später, wenn der Wert des Komponentenverbrauchs zusammen mit dem beendeten FA gebucht wird.</span><span class="sxs-lookup"><span data-stu-id="e7993-109">Without routing link codes, the inventory value will increase as output is posted and then decrease at a later point in time when the value of component consumption is posted together with the finished production order.</span></span>

    <span data-ttu-id="e7993-111">Mit allmählicher Verbrauchsbuchung wird die Verfügbarkeit von Komponenten aktueller, was wichtig ist, um die interne Balance zwischen Bedarf und Vorrat aufrechtzuerhalten.</span><span class="sxs-lookup"><span data-stu-id="e7993-111">With gradual consumption posting, the availability of component items is more up-to-date, which is important to maintain the internal balance between demand and supply.</span></span> <span data-ttu-id="e7993-112">Ohne Verbindungscodenummern glauben möglicherweise andere, die Bedarf für die Komponente haben, dass sie verfügbar ist, solange eine verzögerte Verbrauchsbuchung dafür aussteht.</span><span class="sxs-lookup"><span data-stu-id="e7993-112">Without routing link codes, other demands for the component may believe that it is available as long as it is pending a delayed consumption posting.</span></span>  
- <span data-ttu-id="e7993-113">**Just-in-Time**</span><span class="sxs-lookup"><span data-stu-id="e7993-113">**Just-in-Time**</span></span>

    <span data-ttu-id="e7993-114">Mit der Möglichkeit, Produkte an Debitorenanfragen anzupassen, können Sie Abfall minimieren, indem Sie sicherstellen, dass Arbeits- und Systemänderungen nur eintreten, wenn es erforderlich ist.</span><span class="sxs-lookup"><span data-stu-id="e7993-114">With the ability to customize products to customer requests, you can minimize waste by making sure that work and system changes only occur when it is necessary.</span></span>  

<span data-ttu-id="e7993-115">Sie können dies durch Kombination der Rückwärtsbuchen und der Verbindungscodes erreichen, sodass die Menge, die je Arbeitsgang geleert wird, zur aktuellen Isteffektivität des abgeschlossenen Arbeitsgangs proportional ist.</span><span class="sxs-lookup"><span data-stu-id="e7993-115">You can achieve that by combining backward flushing method and routing link codes so that the quantity that is flushed for each operation is proportional to the actual output of the finished operation.</span></span> <span data-ttu-id="e7993-116">Für Artikel, die mit der Rückwärtsbuchungsmethode erstellt wurden, ist das Standardverhalten, Komponentenverbrauch zu berechnen und zu buchen, wenn Sie den Status eines freigegebenen Fertigungsauftrags in **Erledigt** ändern.</span><span class="sxs-lookup"><span data-stu-id="e7993-116">For items that are set up with backward flushing method, the default behavior is to calculate and post component consumption when you change the status of a released production order to **Finished**.</span></span> <span data-ttu-id="e7993-117">Wenn Sie auch Verbindungscodes definieren, dann erfolgt die Berechnung und Buchung, wenn jeder Arbeitsgang beendet ist, und die Menge, die tatsächlich im Arbeitsgang verbraucht wurde, wird gebucht.</span><span class="sxs-lookup"><span data-stu-id="e7993-117">If you also define routing link codes, then calculation and posting occurs when each operation is finished, and the quantity that was actually consumed in the operation is posted.</span></span> <span data-ttu-id="e7993-118">Weitere Informationen finden Sie unter [Arbeitspläne erstellen](production-how-to-create-routings.md).</span><span class="sxs-lookup"><span data-stu-id="e7993-118">For more information, see [Create Routings](production-how-to-create-routings.md).</span></span>
